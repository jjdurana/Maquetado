Se utiliza para definir un comentario dentro de nuestro archivo .html y no será renderizado en el navegador


<!DOCTYPE>

Define el tipo de archivo a crear, en este caso <!DOCTYPE> HTML


<a>

Define un hipervínculo el cual es usado para poder conectar una página a otra.
Su atributo más importante es el href, el cual indica el destino del vínculo.

<abbr>

Define una abreviación o un acrónimo

Ejemplo:

The WHO was founded in 1948.

<acronym>

No es soportado en HTML5.

Utiliza <abbr> en su lugar.

Define un acrónimo

<address>

Define la información de contacto de el autor/dueño de un documento.

Ejemplo:

Written by Jon Doe.
Visit us at:
Example.com
Box 564, Disneyland
USA

<applet>

No es soportado en HTML5

Usar <embed> o <objet> en su lugar.


<area>

Define un área dentro de una mapa de imagen con áreas donde se puede hacer click


<article>

Especifíca un artículo con contenido autocontenido e independiente.

Un artículo toma prioridad por sí mismo y es posible distribuirlo independientemente del resto de el sitio.

Puede ser usado como un nuevo post de un foro, post de blog o nueva noticia.


<aside>

Define algún contenido a lado del contenido donde es puesto.

Debe ser indirectamente relacionado al contenido circundante.

<audio>

Define contenido de sonido adjunto a un documento, como puede ser música y otros audios vía stream.

Puede contener una o más con diferentes fuentes de audio.

El navegador eligirá la primera fuente que soporte.

Existen tres formatos de audio soportados en HTML: MP3, WAV, OGG.


Ejemplo:

Power Ranger Alert

Click en el botón play para escuchar el llamado de Zordon

<b>

Define el texto en negritas

<base>

Especifica la URL base y/o el objetivo para todas la URL relacionadas en un documento.

Debe tener href o atributo de objetivo presente o ambos.

Sólo puede haber un elemento base en un documento, y debe estar presente en el elemento <head>

<basefont>

No es soportado in HTML5. Usar CSS en su lugar.

Especifica un color por default, tamaño y fuente para todo el texto en el documento.

<bdi>

Aísla una parte del texto que podría tener un formato diferente al de otro texto fuera de ella.

Este elemento es útil cuando se incrusta contenido generado por el usuario con una dirección de texto desconocida.


<bdo>

BDO son las siglas de Bi-Directional Override.

La etiqueta se utiliza para anular la dirección actual del texto.

Ejemplo:

Este parrafo irá de izquierda a derecha.

Este párrafo irá de derecha a izquierda.

<big>

No es soportado in HTML5. Usar CSS en su lugar.

Define texto en grande

<blockquote>

Define que una sección es una cita de otra fuente.

Ejemplo:

Aquí una cita desde el sitio del WWF

For 50 years, WWF has been protecting the future of nature. The world's leading conservation organization, WWF works in 100 countries and is supported by 1.2 million members in the United States and close to 5 million globally.

<body>

Define el cuerpo del documento.

<br>

Define el salto de una línea

<button>

Define un botón clickeable.

Ejemplo:

<canvas>

Se utiliza para dibujar gráficos, sobre la marcha, a través de secuencias de comandos

La etiqueta <canvas> es transparente y es solo un contenedor para gráficos, debe usar un script para dibujar los gráficos.

Cualquier texto dentro del elemento <canvas> se mostrará en los navegadores con JavaScript deshabilitado y en los navegadores que no admitan <canvas>.

Ejemplo:

<caption>

La etiqueta <caption> define un título de tabla.

La etiqueta <caption> debe insertarse inmediatamente después de la etiqueta <table>.

Consejo: De forma predeterminada, el título de una tabla se alineará en el centro sobre una tabla.

Sin embargo, las propiedades CSS text-align y caption-side se pueden usar para alinear y colocar el título.

Ejemplo:

Monthly savings
Month Savings
January $100

<center>

No es soportado in HTML5. Usar CSS en su lugar.

Define el centrado de un texto.

<cite>

La etiqueta <cite> define el título de una obra creativa (por ejemplo, un libro, un poema, una canción, una película, una pintura, una escultura, etc.).

Nota: El nombre de una persona no es el título de una obra.

El texto del elemento <cite> suele estar en cursiva.

Ejemplo:

The Scream by Edward Munch. Painted in 1893.

<code>

La etiqueta <code> se utiliza para definir un fragmento de código informático.

El contenido del interior se muestra en la fuente monoespaciada predeterminada del navegador.

Consejo: Esta etiqueta no está obsoleta. Sin embargo, es posible lograr un efecto más rico mediante el uso de CSS.

Ejemplo:

El elemento code

The HTML button tag defines a clickable button.

The CSS background-color property defines the background color of an element.

<col>

La etiqueta <col> especifica las propiedades de columna de cada columna dentro de un elemento <colgroup>.

La etiqueta <col> es útil para aplicar estilos a columnas enteras, en lugar de repetir los estilos para cada celda, para cada fila.

Ejemplo:

ISBN Title Price
3476896 My first HTML $53

<colgroup>

La etiqueta <colgroup> especifica un grupo de una o varias columnas de una tabla para su formato.

La etiqueta <colgroup> es útil para aplicar estilos a columnas enteras, en lugar de repetir los estilos para cada celda, para cada fila.

Nota: La etiqueta <colgroup> debe ser un elemento secundario de un elemento <table>, después de cualquier elemento <caption> y antes de cualquier elemento <thead>, <tbody>, <tfoot> y <tr>.

Sugerencia: Para definir diferentes propiedades de una columna dentro de un <colgroup>, utilice la etiqueta <col> dentro de la etiqueta <colgroup>.

<data>

La etiqueta <data> se utiliza para agregar una traducción legible por máquina de un contenido determinado.

Este elemento proporciona un valor legible por máquina para los procesadores de datos y un valor legible por humanos para la representación en un explorador.

Consejo: Si el contenido está relacionado con la hora o la fecha, utilice el elemento <time> en su lugar.

Ejemplo:

  • Cherry Tomato
  • Beef Tomato
  • Snack Tomato

<datalist>

La etiqueta <datalist> especifica una lista de opciones predefinidas para un elemento <input>.

La etiqueta <datalist> se utiliza para proporcionar una función de "autocompletar" para los elementos <input>.

Los usuarios verán una lista desplegable de opciones predefinidas a medida que ingresan datos.

El atributo id del elemento <datalist> debe ser igual al atributo list del elemento <input> (esto los une).

Ejemplo:

<dd>

La etiqueta <dd> se utiliza para describir un término/nombre en una lista de descripción.

La etiqueta <dd> se utiliza junto con <dl> (define una lista de descripción) y <dt> (define términos/nombres).

Dentro de una etiqueta <dd> puedes poner párrafos, saltos de línea, imágenes, enlaces, listas, etc.

Ejemplo:

These three elements are used to create a description list:

Coffee
Black hot drink
Milk
White cold drink

<dl>

La etiqueta <del> define el texto que se ha eliminado de un documento. Por lo general, los navegadores trazan una línea a través del texto eliminado.

Ejemplo:

My favorite color is blue red!

<details>

La etiqueta <details> especifica detalles adicionales que el usuario puede abrir y cerrar a petición.

La etiqueta <details> se utiliza a menudo para crear un widget interactivo que el usuario puede abrir y cerrar. De forma predeterminada, el widget está cerrado. Cuando se abre, se expande y muestra el contenido que contiene.

Cualquier tipo de contenido se puede poner dentro de la etiqueta <details>.

Consejo: La etiqueta <summary> se utiliza junto con <details> para especificar un encabezado visible para los detalles.

Ejemplo:

Epcot Center

Epcot is a theme park at Walt Disney World Resort featuring exciting attractions, international pavilions, award-winning fireworks and seasonal special events.

<dfn>

La etiqueta <dfn> representa el "elemento de definición" y especifica un término que se va a definir dentro del contenido.

El elemento primario más cercano de la etiqueta <dfn> también debe contener la definición/explicación del término.

El término dentro de la etiqueta <dfn> puede ser cualquiera de los siguientes:

Al igual que el contenido del elemento <dfn>:

HTML is the standard markup language for creating web pages.

O bien, con el atributo title agregado:

HTML is the standard markup language for creating web pages.

O bien, con una etiqueta <abbr> dentro del elemento <dfn>:

HTML is the standard markup language for creating web pages.

O bien, con el atributo id agregado. Luego, cada vez que se usa un término, puede referirse a la definición con una etiqueta <a>:

HTML is the standard markup language for creating web pages.

This is some text...

This is some text...

Learn HTML now.

<dialog>

La etiqueta <dialog> define un cuadro de diálogo o una subventana.

El elemento <dialog> facilita la creación de cuadros de diálogo emergentes y modales en una página web.

Ejemplo:
This is an open dialog window

<dir>

No es compatible con HTML5. Use <ul> en su lugar.

Define una lista de directorios

<div>

La etiqueta <div> define una división o una sección en un documento HTML.

La etiqueta se utiliza como contenedor para elementos HTML, que luego se diseña con CSS o se manipula con JavaScript.

La etiqueta <div> se diseña fácilmente mediante el atributo class o id.

Cualquier tipo de contenido se puede poner dentro de la etiqueta <div>

Nota: De forma predeterminada, los navegadores siempre colocan un salto de línea antes y después del elemento <div>.

<dl>

La etiqueta <dl> define una lista de descripciones.

La etiqueta <dl> se utiliza junto con <dt> (define términos/nombres) y <dd> (describe cada término/nombre).

Ejemplo:

Coffee
Black hot drink
Milk
White cold drink

<dt>

La etiqueta <dt> define un término/nombre en una lista de descripciones.

La etiqueta <dt> se utiliza junto con <dl> (define una lista de descripciones) y <dd> (describe cada término/nombre).

Ejemplo:

Coffee
Black hot drink
Milk
White cold drink

<em>

La etiqueta <em> se utiliza para definir el texto enfatizado. El contenido del interior suele mostrarse en cursiva.

Un lector de pantalla pronunciará las palabras en <em> con énfasis, utilizando el acento verbal.

Ejemplo:

You have to hurry up!

We cannot live like this.

<embed>

La etiqueta <embed> define un contenedor para un recurso externo, como una página web, una imagen, un reproductor multimedia o una aplicación de complemento.

Ejemplo

<fieldset>

La etiqueta <fieldset> se utiliza para agrupar elementos relacionados en un formulario.

Dibuja un cuadro alrededor de los elementos relacionados.

Ejemplo:

Personalia:







<figcaption>

La etiqueta <figcaption> define un título para un elemento <figure>.

El elemento <figcaption> se puede colocar como el primer o el último elemento secundario del elemento <figure>.

Ejemplo:

<figure>

La etiqueta <figure> especifica contenido autónomo, como ilustraciones, diagramas, fotos, listados de códigos, etc.

Si bien el contenido del elemento <figure> está relacionado con el flujo principal, su posición es independiente del flujo principal y, si se elimina, no debería afectar el flujo del documento.

Consejo: El elemento <figcaption> se utiliza para añadir un título para el elemento <figure>.
