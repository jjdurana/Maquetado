Existen tres formatos de audio soportados en HTML: MP3, WAV, OGG.

<cite>

La etiqueta <cite> define el título de una obra creativa (por ejemplo, un libro, un poema, una canción, una película, una pintura, una escultura, etc.).

Nota: El nombre de una persona no es el título de una obra.

El texto del elemento <cite> suele estar en cursiva.

Ejemplo:

La etiqueta <code> se utiliza para definir un fragmento de código informático.

El contenido del interior se muestra en la fuente monoespaciada predeterminada del navegador.

Consejo: Esta etiqueta no está obsoleta. Sin embargo, es posible lograr un efecto más rico mediante el uso de CSS.

<col>

La etiqueta <col> especifica las propiedades de columna de cada columna dentro de un elemento <colgroup>.

La etiqueta <col> es útil para aplicar estilos a columnas enteras, en lugar de repetir los estilos para cada celda, para cada fila.

Ejemplo:

<colgroup>

La etiqueta <colgroup> especifica un grupo de una o varias columnas de una tabla para su formato.

La etiqueta <colgroup> es útil para aplicar estilos a columnas enteras, en lugar de repetir los estilos para cada celda, para cada fila.

Nota: La etiqueta <colgroup> debe ser un elemento secundario de un elemento <table>, después de cualquier elemento <caption> y antes de cualquier elemento <thead>, <tbody>, <tfoot> y <tr>.

Sugerencia: Para definir diferentes propiedades de una columna dentro de un <colgroup>, utilice la etiqueta <col> dentro de la etiqueta <colgroup>.

La etiqueta <data> se utiliza para agregar una traducción legible por máquina de un contenido determinado.

Este elemento proporciona un valor legible por máquina para los procesadores de datos y un valor legible por humanos para la representación en un explorador.

Consejo: Si el contenido está relacionado con la hora o la fecha, utilice el elemento <time> en su lugar.

Los usuarios verán una lista desplegable de opciones predefinidas a medida que ingresan datos.

El atributo id del elemento <datalist> debe ser igual al atributo list del elemento <input> (esto los une).

La etiqueta <details> especifica detalles adicionales que el usuario puede abrir y cerrar a petición.

La etiqueta <details> se utiliza a menudo para crear un widget interactivo que el usuario puede abrir y cerrar. De forma predeterminada, el widget está cerrado. Cuando se abre, se expande y muestra el contenido que contiene.

Cualquier tipo de contenido se puede poner dentro de la etiqueta <details>.

<dfn>

La etiqueta <dfn> representa el "elemento de definición" y especifica un término que se va a definir dentro del contenido.

El elemento primario más cercano de la etiqueta <dfn> también debe contener la definición/explicación del término.

El término dentro de la etiqueta <dfn> puede ser cualquiera de los siguientes:

Al igual que el contenido del elemento <dfn>:

HTML is the standard markup language for creating web pages.

O bien, con el atributo title agregado:

HTML is the standard markup language for creating web pages.

O bien, con una etiqueta <abbr> dentro del elemento <dfn>:

HTML is the standard markup language for creating web pages.

O bien, con el atributo id agregado. Luego, cada vez que se usa un término, puede referirse a la definición con una etiqueta <a>:

HTML is the standard markup language for creating web pages.

This is some text...

This is some text...

Learn HTML now.

<head>

El elemento <head> es un contenedor de metadatos (datos sobre datos) y se coloca entre la etiqueta <html> y la etiqueta <body>.

Los metadatos son datos sobre el documento HTML. No se muestran los metadatos.

Los metadatos suelen definir el título del documento, el conjunto de caracteres, los estilos, los guiones y otra metainformación.

Los siguientes elementos pueden ir dentro del elemento <head>:

  • <title> (requerido en cada archivo HTML)
  • style
  • base
  • link
  • meta
  • script
  • noscript

<header>

El elemento <header> representa un contenedor para contenido introductorio o un conjunto de vínculos de navegación.

Un elemento <header> normalmente contiene:

  • uno o más elementos de encabezado (<h1> - <h6>)
  • Logotipo o icono
  • Información de autoría

Nota: Puede tener varios elementos <header> en un documento HTML. Sin embargo, <header> no se puede colocar dentro de un <footer>, <address> u otro elemento <header>.

Los principales idiomas de la Web

HTML es el lenguaje de marcado estándar para crear páginas web. HTML describe la estructura de una página web y consta de una serie de elementos. Los elementos HTML le dicen al navegador cómo mostrar el contenido.

CSS es un lenguaje que describe cómo se deben mostrar los elementos HTML en la pantalla, el papel o en otros medios. CSS ahorra mucho trabajo, ya que puede controlar el diseño de varias páginas web a la vez.

<html

La etiqueta <html> representa la raíz de un documento HTML.

La etiqueta <html> es el contenedor de todos los demás elementos HTML (excepto la etiqueta <! DOCTYPE> ).

Nota: Siempre debe incluir el atributo lang dentro de la etiqueta <html> para declarar el idioma de la página web. Esto está destinado a ayudar a los motores de búsqueda y navegadores.

<i>

La etiqueta <i> define una parte del texto en una voz o estado de ánimo alternativo. El contenido del interior suele mostrarse en cursiva.

La etiqueta <i> se utiliza a menudo para indicar un término técnico, una frase de otro idioma, un pensamiento, el nombre de un barco, etc.

Utilice el elemento <i> solo cuando no haya un elemento semántico más apropiado, como:

  • <em> (texto subrayado)
  • <strong> (texto importante)
  • <mark> (texto marcado/resaltado)
  • <cite> (título de una obra)
  • <dfn> (un término de definición)

<img>

La etiqueta <img> se utiliza para incrustar una imagen en una página HTML.

Las imágenes no se insertan técnicamente en una página web; Las imágenes están vinculadas a páginas web. La etiqueta <img> crea un espacio de retención para la imagen a la que se hace referencia.

La etiqueta <img> tiene dos atributos obligatorios:

src - Especifica la ruta a la imagen

alt - Especifica un texto alternativo para la imagen, si la imagen por alguna razón no se puede mostrar

Nota: Además, especifique siempre la anchura y la altura de una imagen. Si no se especifican la anchura y la altura, es posible que la página parpadee mientras se carga la imagen.

Consejo: Para vincular una imagen a otro documento, basta con anidar la etiqueta <img> dentro de una etiqueta <a> </a>

La etiqueta <link> se usa con mayor frecuencia para vincular a hojas de estilo externas o para agregar un favicon a su sitio web.

El elemento <link> es un elemento vacío, solo contiene atributos.

<main>

La etiqueta <main> especifica el contenido principal de un documento.

El contenido dentro del elemento <principal> debe ser único para el documento. No debe contener ningún contenido que se repita en los documentos, como barras laterales, enlaces de navegación, información de derechos de autor, logotipos de sitios y formularios de búsqueda.

Nota: No debe haber más de un elemento <main> en un documento. El elemento <main> NO debe ser descendiente de un elemento <article>, <aside>, <footer>, <header> o <nav>.

<map>

La etiqueta <map> se utiliza para definir un mapa de imagen. Un mapa de imagen es una imagen con áreas en las que se puede hacer clic.

El atributo name del elemento <map> se asocia con el atributo de <img> y crea una relación entre la imagen y el mapa.

El elemento <map> contiene una serie de elementos <area> que definen las áreas en las que se puede hacer clic en el mapa de imagen.

<meta>

La etiqueta <meta> define los metadatos de un documento HTML. Los metadatos son datos (información) sobre los datos.

Siempre van dentro del elemento <head> y normalmente se utilizan para especificar el conjunto de caracteres, la descripción de la página, las palabras clave, el autor del documento y la configuración de la ventana gráfica.

Los metadatos no se mostrarán en la página, pero se pueden analizar automáticamente.

Los metadatos son utilizados por los navegadores (cómo mostrar contenido o volver a cargar la página), los motores de búsqueda (palabras clave) y otros servicios web.

Existe un método para permitir que los diseñadores web tomen el control de la ventana gráfica (el área visible del usuario de una página web), a través de la etiqueta <meta>.

<nav>

La etiqueta <nav> define un conjunto de vínculos de navegación.

Tenga en cuenta que NO todos los enlaces de un documento deben estar dentro de un elemento <nav>. El elemento <nav> está pensado solo para bloques principales de enlaces de navegación.

Los navegadores, como los lectores de pantalla para usuarios deshabilitados, pueden usar este elemento para determinar si se omite la representación inicial de este contenido.

El elemento <noscript> se puede utilizar tanto en <head> como en <body>. Cuando se usa dentro de <head>, el elemento <noscript> solo puede contener elementos <link>, <style> y <meta>.

<picture>

La etiqueta <picture> ofrece a los desarrolladores web más flexibilidad a la hora de especificar los recursos de imagen.

El uso más común del elemento <imagen> será para la dirección de arte en diseños responsivos. En lugar de tener una imagen que se escala hacia arriba o hacia abajo en función del ancho de la ventanilla, se pueden diseñar varias imágenes para llenar mejor la ventanilla del navegador.

El elemento <picture> contiene dos etiquetas: una o varias etiquetas <source> y una etiqueta <img>.

El navegador buscará el primer elemento <source> en el que la consulta de medios coincida con el ancho de la ventana gráfica actual y, a continuación, mostrará la imagen adecuada (especificada en el atributo srcset). El elemento <img> es necesario como último elemento secundario del elemento <picture>, como opción de reserva si ninguna de las etiquetas de origen coincide.

Consejo: El elemento <imagen> funciona de forma "similar" a <vídeo> y <audio>. Configura diferentes fuentes, y la primera fuente que se ajusta a las preferencias es la que se está utilizando.

El elemento <tr> define una fila de tabla, el elemento <th> define un encabezado de tabla y el elemento <td> define una celda de tabla.

<tbody>

La etiqueta <tbody> se utiliza para agrupar el contenido del cuerpo en una tabla HTML.

El elemento <tbody> se utiliza junto con los elementos <thead> y <tfoot> para especificar cada parte de una tabla (cuerpo, encabezado, pie de página).

Los navegadores pueden usar estos elementos para habilitar el desplazamiento del cuerpo de la tabla independientemente del encabezado y el pie de página. Además, al imprimir una tabla grande que abarca varias páginas, estos elementos pueden permitir que el encabezado y el pie de página de la tabla se impriman en la parte superior e inferior de cada página.

Nota: El elemento <tbody> debe tener una o más etiquetas <tr> en su interior.

La etiqueta <tbody> se debe utilizar en el siguiente contexto: como elemento secundario de un elemento <table>, después de cualquier elemento <caption>, <colgroup> y <thead>.

Sugerencia: Los elementos <thead>, <tbody> y <tfoot> no afectarán al diseño de la tabla de forma predeterminada. Sin embargo, puedes usar CSS para dar estilo a estos elementos.

<title>

La etiqueta <title> define el título del documento. El título debe ser solo texto y se muestra en la barra de título del navegador o en la pestaña de la página.

La etiqueta <title> es obligatoria en los documentos HTML.

¡El contenido del título de una página es muy importante para la optimización de motores de búsqueda (SEO)! El título de la página es utilizado por los algoritmos de los motores de búsqueda para decidir el orden al enumerar las páginas en los resultados de búsqueda.

El elemento <title>:

  • Define un título en la barra de herramientas del navegador
  • Proporciona un título para la página cuando se agrega a Favoritos
  • Muestra un título para la página en los resultados del motor de búsqueda

Aquí hay algunos consejos para crear buenos títulos:

  • Opta por un título más largo y descriptivo (evita los títulos de una o dos palabras)
  • Los motores de búsqueda mostrarán entre 50 y 60 caracteres del título, así que trata de no tener títulos más largos que eso
  • No utilices solo una lista de palabras como título (esto puede reducir la posición de la página en los resultados de búsqueda)
  • Por lo tanto, ¡intenta que el título sea lo más preciso y significativo posible!

Nota: NO puede tener más de un elemento <title> en un documento HTML.
